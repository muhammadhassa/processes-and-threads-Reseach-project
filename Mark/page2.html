Advantages of Processes

In this section, I'm going to discuss about the advanatges of processes compared to threads.

  • One advantage of processes is the fact that each process has it's own memory space separate from other processes with each process consisting of code text, data containing global variables, a stack that consists of local variables for function parameters, running procedure calls and return addresses, and a heap which consists of dynamically allocated during run time. If one process corrupted it's own memory space, the other processes will not be affected.


  • Using processes allows each computation with it's own virtual machine. So each computation is independent.

  • Threads must run only on the same machine while processes can run on different machines.

Disadvantages of Processes

In this section, I'm going to discuss the disadvanatages of processes

  • Processes are expensive in terms of resource consumption. Because of this, processes takes up memory.

  • Processes are also expensive in terms of time consumption. Because of this, processes needs more time for termination.

  • Processes needs more time for creation which is the making of children processes.[1]

  • Processes take more time in context switching between different processes even if they all belong to the same operating system.
