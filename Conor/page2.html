Various States of a Process

1. New
A new process is one which is about to be picked up by the Operating System into the main memory.

2. Ready
After a process is created, it enters the ready state in which it is waiting for the CPU to be assigned.The OS will move the processes from secondary memory into main memory. Processes which are residing in the main memory and waiting to be assigned are said to be ready state processes. At any given time, there may be numerous processes waiting to be executed while present in the ready state.

3. Running
Depending on whichever scheduling algorithm is being utilised, one of the ready state processes will be selected by the OS. If we have a single CPU in our system, only one of our processes can be run at any given time. If we have to access several processors, the equivalent number of processes can operate in the running state concurrently.

4. Block or wait
Depending on the scheduling algorithm or the behaviour of the process, the running state may transition into a block or waiting state. The process then waits for input from the user or to be assigned a certain resource, meanwhile the CPU can be assigned to other processes.

5. Completion or termination
The process which has finished its execution enters the completion state. At this point all the Process Control Block information saved on the process can be discarded and the OS can terminate the process.

6. Suspend ready
A process in the ready state which needs to be moved back to secondary memory is said to be in the suspend ready state. This happens when there is a lack of necessary resources for the process to proceed, most often a lack of primary memory. If the main memory is full, it may push back a process to free up memory and make way for a higher priority process to execute.

7. Suspend wait
Rather than suspending a ready process to make way for higher priority process, it is better to push a waiting process back to secondary memory. This is preferable as the waiting process is already idle while it waits for certain necessary resources to become available. Once these resources are available and the waiting state has finished, the waiting process can resume being executed by main memory.
